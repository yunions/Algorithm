 # 04. Stack & Queue

2021.06.20

- Stack, Queue : 기본적인 자료구조 중 하나 → 데이터를 넣곡 뽑는 방향이 다름

## (1) Stack

: 데이터를 임시 저장할 때 사용하는 자료구조

- 데이터의 입력과 출력 순서 :  **LIFO**(last in first out, 후입선출방식)  like ctrl + z

### Stack basic

- **stk** : 스택 배열(list)
- **capacity** : 스택의 최대 크기(int) = len(stk)
- **ptr** : 스택 포인터, 스택에 쌓여 있는 데이터의 수

### Stack function

- **push** : 가장 **top**에 데이터를 넣음
- **pop** : 가장 **top**에서 데이터를 꺼냄
- **peak** : top의 데이터 값 확인(= 가장 최근 push 데이터 확인)





## (2) Queue

- 데이터의 입력과 출력 순서 : **FIFO**(First in first out, 선입선출방식)

  → stack구조와 데이터가 들어오고 나가는 위치가 다름

### Queue fuction

- **push** : 가장 **top**에 데이터를 넣음
- **pop** : 가장 **bottom**에서 데이터를 꺼냄

### Ring Buffer를 활용한 Queue

: enqueue는 append 하기 쉽지만, dequeue할 때 index를 전부 변경해야함

- **Ring Buffer**

: 배열 맨 끝의 원소와 배열 맨 앞의 원소가 연결되는 자료구조 형태
